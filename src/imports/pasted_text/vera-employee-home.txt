Create a high-fidelity Figma design concept for a mobile-first web app called **VERA**.

Important: this is **not a landing page** and not a generic mobile app mockup. It is a **mobile web app / PWA-style operational interface** embedded in a website, designed for frontline employees and managers to create, review, and control product write-off requests. The app should feel like a modern business tool: fast, soft, readable, trustworthy, and slightly playful — but never childish.

## Product context

VERA is a voice-first write-off system for trade points. Employees can report damaged or unusable products by voice. AI turns their speech into structured write-off data, asks follow-up questions if information is missing, generates a clean professional comment, attaches photo proof, and sends the request for review. Managers/reviewers can see all write-off requests in a clean smart table/card system, approve or reject them, and see Iiko sync status.

Core flow:
Employee speaks → AI extracts data → missing fields are clarified → photo proof is attached → request is submitted → reviewer checks → approved request syncs to Iiko.

## Brand direction

Use the provided VERA logo concept as the visual reference:

* Soft rounded horizontal badge
* Strawberry-red / raspberry palette
* Creamy text
* Slight “bitten” corner detail
* Soft floral/chocolate-like feeling
* Digitalized 2D style, not heavy 3D clay
* Friendly, warm, modern, readable

The app should feel like the logo became a full interface system.

## Anti-reference

Avoid old enterprise software aesthetics:

* no 1C-style dense tables
* no gray corporate clutter
* no outdated admin dashboard look
* no tiny unreadable fields
* no harsh borders everywhere
* no heavy Excel-like tables
* no generic SaaS purple/blue gradient
* no “AI slop” cards with meaningless glowing blobs
* no fake futuristic cyberpunk
* no overused glassmorphism everywhere
* no random decorative 3D objects

The interface should feel like a new generation business app: soft, operational, tactile, calm, fast, and clear.

## Visual style

Design style:

* 2D / semi-flat digital style
* soft rounded geometry
* subtle depth only where useful
* very soft shadows, not heavy neumorphism
* smooth surfaces
* clean hierarchy
* generous spacing
* readable cards
* warm, modern, slightly playful details
* business-grade usability

The UI should feel warm and human, but still precise and professional.

Think:

* modern PWA
* Linear-level clarity
* iiko but much softer and more modern
* Notion/Arc-like simplicity
* gentle food-tech warmth
* strawberry cream interface
* operational tool, not toy

## Layout requirements

Design for mobile web first:

* Frame size: 390 × 844 or 393 × 852
* Safe mobile spacing
* Bottom navigation or compact top navigation
* Thumb-friendly controls
* Large primary buttons
* Inputs should be easy to tap
* Cards should be readable at a glance
* Avoid desktop-like density on mobile

Also create one wider responsive frame for reviewer/manager view:

* Tablet / desktop web app frame: 1200 × 800
* This should show smart write-off queue/table in a modern readable way

## Motion and interaction style

Add notes for soft animations and transitions throughout the design:

* Screen transitions: soft slide + fade, 220–320ms
* Cards appear with gentle upward motion
* Voice recording has a soft pulsing strawberry waveform
* AI extraction fields fill in one by one
* Missing field prompts appear as soft chips/cards
* Submit button has gentle press feedback
* Approval action has a calm success animation
* Iiko sync status changes with a smooth progress-to-check transition
* No aggressive bouncy animations
* No childish confetti
* Motion should feel premium, calm, and operational

## Microcopy tone

Tone should be:

* clear
* warm
* operational
* not overly cute
* not corporate-boring

Examples:

* “Tell VERA what happened”
* “I found most details”
* “Two details are missing”
* “Attach photo proof”
* “Ready for review”
* “Sent to manager”
* “Synced to Iiko”
* “Photo is unclear”
* “Check before approval”

Avoid:

* “Magic AI”
* “Supercharge your workflow”
* “Revolutionary”
* “Your intelligent companion”
* generic AI marketing phrases

## AI behavior in UI

Make AI feel like a quiet assistant, not the whole product.

AI should:

* extract fields
* clean the comment
* ask missing questions
* flag uncertainty
* help the reviewer read faster

AI should not:

* approve requests automatically
* dominate the interface
* look like a chatbot
* create futuristic clutter
